# Интеграция с Telegram для уведомлений

## Обзор

Интеграция с Telegram предоставляет возможность отправки уведомлений пользователям через Telegram Bot. Система имеет следующие компоненты:

1. **Backend-сервер Node.js** (`server.cjs`) — обрабатывает API-запросы для отправки уведомлений
2. **Telegram-бот** — принимает команды от пользователей и доставляет уведомления
3. **Таблицы в Supabase** — хранят токены и привязки Telegram-аккаунтов
4. **Клиентский сервис** — позволяет фронтенду отправлять уведомления

## Настройка окружения

### 1. Таблицы в базе данных Supabase

Необходимы две таблицы:

**telegram_link_tokens**
- `id` — UUID, первичный ключ
- `user_id` — UUID, ID пользователя из auth.users
- `token` — текст, уникальный токен для привязки
- `created_at` — timestamp, время создания
- `used` — boolean, использован ли токен

**user_telegram_accounts**
- `id` — UUID, первичный ключ
- `user_id` — UUID, ID пользователя из auth.users
- `telegram_user_id` — bigint, ID пользователя в Telegram
- `username` — текст, имя пользователя в Telegram
- `created_at` — timestamp, время привязки

### 2. Запуск серверов

1. Запустите backend-сервер:
   ```
   node server.cjs
   ```

2. Убедитесь, что Telegram-бот запущен и доступен.

## Использование в коде

### Привязка Telegram-аккаунта

Для привязки аккаунта пользователя:

```typescript
// Получение ссылки для привязки
const response = await fetch('http://localhost:3001/api/telegram/generate-token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    userId: 'id-пользователя'
  })
});

const { link } = await response.json();
// Предоставьте ссылку пользователю (например: https://t.me/ugrabuilders_bot?start=token)
```

### Получение привязанных аккаунтов

```typescript
const response = await fetch(`http://localhost:3001/api/telegram/accounts?userId=id-пользователя`);
const { accounts } = await response.json();
// accounts — массив с привязанными аккаунтами
```

### Отвязка аккаунта

```typescript
await fetch('http://localhost:3001/api/telegram/unlink', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    userId: 'id-пользователя',
    telegram_user_id: 'id-в-telegram'
  })
});
```

## Отправка уведомлений

Для отправки уведомлений используйте сервис уведомлений:

```typescript
import notificationService from "@/services/notifications/notificationService";

// Уведомление о смене статуса проекта
await notificationService.notifyProjectStatusChange(
  userId,        // ID пользователя
  projectId,     // ID проекта
  projectTitle,  // Название проекта
  status         // Новый статус
);

// Уведомление о смене этапа продажи
await notificationService.notifyClientSaleStageChange(
  userId,  // ID пользователя-клиента
  stage    // Новый этап продажи
);

// Уведомление о смене этапа строительства
await notificationService.notifyConstructionStageChange(
  userId,  // ID пользователя-клиента
  stage    // Новый этап строительства
);

// Уведомление о новой задаче
await notificationService.notifyNewTask(
  userId,  // ID пользователя
  {
    task: 'Название задачи',
    date: '2023-12-01',
    assignedTo: 'Ответственный'
  }
);

// Уведомление о загрузке документа
await notificationService.notifyDocumentUploaded(
  userId,    // ID пользователя
  fileName,  // Имя файла
  fileType   // Тип файла ('documents', 'contracts')
);

// Произвольное уведомление
await notificationService.sendCustomNotification(
  userId,  // ID пользователя
  title,   // Заголовок уведомления
  body     // Текст уведомления
);
```

## Безопасность

- API-запросы используют сервисный ключ Supabase для доступа к данным
- Токены для привязки действительны только 10 минут и могут быть использованы только один раз
- Сервер проверяет права доступа для всех операций

## Устранение неполадок

1. Если уведомления не отправляются:
   - Проверьте работу backend-сервера (`node server.cjs`)
   - Убедитесь, что Telegram-бот активен
   - Проверьте наличие привязанных аккаунтов в таблице `user_telegram_accounts`

2. Если привязка не работает:
   - Проверьте правильность генерации и использования токенов
   - Убедитесь, что пользователь правильно выполняет команду `/start token` в боте 

3. Проблема с распознаванием привязанных аккаунтов:
   - Проверьте соответствие полей в коде бэкенда (api/telegram.cjs) и структуры таблицы user_profiles
   - Убедитесь, что запрашиваются только существующие поля из таблицы
   - Проверьте логи сервера на наличие ошибок при выполнении SQL-запросов
   - Проверьте, что данные о пользователе присутствуют в таблице user_profiles

### Решение проблемы с приветствием

Если после привязки аккаунта бот все равно приветствует пользователя как незнакомого, проверьте:

1. **Корректность SQL-запросов**:
   - В функции `checkTelegramAccount` должны запрашиваться только существующие поля таблицы user_profiles
   - Поля, которые гарантированно существуют: `id`, `username`, `role` 

2. **Обработка отсутствующих данных**:
   - Код бота должен корректно обрабатывать случаи, когда поля username или другие отсутствуют
   - Для этого используется резервный вариант с именем "пользователь" или ID пользователя

3. **Отладочная информация**:
   - В текущей версии бота расширена отладка для диагностики проблем
   - В логах должны быть видны структуры данных на каждом этапе обработки запроса

## Функциональность бота

### Команды бота

1. `/start` - Начальное приветствие:
   - Если аккаунт привязан, приветствует пользователя по имени и показывает доступные команды
   - Если аккаунт не привязан, предлагает произвести привязку через личный кабинет
   - При указании токена (например, `/start token123`) производит привязку аккаунта

2. `/help` - Справка по боту:
   - Показывает список всех доступных команд и краткое описание бота

3. `/status` - Проверка статуса (в разработке):
   - В будущем будет показывать текущий статус ваших проектов и задач

4. `/notify_off` - Отключение уведомлений (в разработке):
   - В будущем позволит временно отключить уведомления

5. `/notify_on` - Включение уведомлений (в разработке):
   - В будущем позволит восстановить отправку уведомлений

### Привязка к аккаунту

При первом использовании бота пользователь увидит общее приветствие и инструкцию о привязке.
После привязки аккаунта через личный кабинет, бот будет узнавать пользователя и предоставлять
персонализированную информацию.
